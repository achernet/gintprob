1. Customers

    1. As a customer, I need to be able to see what reservations I have booked.

      1. If I want to cancel a reservation, it should ask me to confirm the cancellation.

        1. After confirmation, the UI should immediately reflect the change.

      2. I might need to add someone to the reservation. Based on the table configuration (that only Managers should see directly), if the tables are “maxed out”, the UI for adding people should get grayed out.

      3. I might need to make the reservation later or earlier. Based on the table configuration at the desired time, is this possible?

        1. If the kitchen closes an hour before the rest of the restaurant, is this still possible?

    2. As a customer, I need to be able to search through local restaurant listings based on a variety of criteria. (Technically out of scope with respect to the reservation system itself, but definitely in scope in the sense of customer retention, market share, and other ROI)

      1. I would like to avoid places that are too expensive.

      2. I would like to also avoid places where lots of other customers are kvetching and saying that it's a ripoff.

      3. I would like to avoid places that are mediocre in general.

      4. I would like to support local establishments.

      5. If I'm allergic to {cilantro|peanuts|gluten} will I still be able to eat here?

  2. Managers

    1. Managers need to be able to adjust their table configurations easily in order to accommodate reservation changes, which can happen quite frequently, especially if the sizes of the parties involved are larger than 5 or 6.

    2. The average time spent by a customer at a table is an essential metric since it dictates the time slot needed for a given reservation in the database.
